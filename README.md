# 🌾 AgriSearch — Agriculture Information System
### Victoria, Laguna Edition

A full-featured web-based agricultural information system for Victoria, Laguna. Built with Flask + MySQL, it supports farm & livestock management, crop tracking, supply distribution, animal health monitoring, a live GIS map with barangay boundaries, and an AI chatbot assistant.

---

## ✨ Features

### 🔐 Authentication & Roles
- Secure registration, login, password reset (token-based)
- 5-tier role system: **Farmer**, **Barangay Officer**, **Municipal Coordinator**, **Victoria Admin**, **System Admin**
- Role-based dashboards and protected routes

### 🏡 Farm Management
- Create, edit, and view farm profiles with GPS coordinates
- Barangay assignment from 9 Victoria barangays
- Map picker for GPS coordinates (Leaflet + HTTPS toggle)

### 🐄 Livestock Management
- Add livestock records per farm (species, breed, count)
- Track health status

### 🌱 Crop Management
- Log crop types, planting dates, and growth records
- Monitor crop status per farm

### 📦 Supply & Distribution
- Manage agriculture supply inventory
- Distribution cycles with farmer requests and officer approval flow
- Farm-level supply usage history

### 💉 Animal Health
- Disease and vaccine tracking
- Health alerts dashboard
- Vaccine inventory management

### 🗺️ GIS Farm Map
- Interactive Leaflet map with all farm markers
- Real OSM barangay boundaries (Shapely polygonize, pre-generated)
- Boundary toggle, heatmap, satellite layer, barangay labels
- Radius-based farm search (click-to-place circle)
- Barangay info cards with per-barangay farm statistics
- Mobile-responsive: slide-in filter panel, tap-away overlay

### 🤖 AgriBot Chatbot
- Powered by **Groq API** (`llama-3.3-70b-versatile`)
- **Persistent conversation history** (ChatGPT-style)
- Left sidebar with conversation list, timestamps, and delete
- Auto-titles conversations from the first message
- Stores full history in MySQL (`chat_conversations`, `chat_messages`)
- Suggestion chips for quick prompts
- Floating bubble widget on all pages (logged-in users only)

### 📊 Dashboards
- Role-specific dashboards with statistics
- Activity timelines, quick actions, admin panels

## 🚀 Quick Start

### 1. Prerequisites
- **Python 3.8+**
- **MySQL** running locally (e.g. XAMPP, MySQL Workbench, or plain MySQL)

### 2. Configure `.env`
Create a `.env` file in the project root (copy the template below):
```env
# Flask
FLASK_APP=run.py
FLASK_ENV=development
SECRET_KEY=change-this-to-a-random-secret

# MySQL Database
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=agrisearch

# Server
API_PORT=5000

# HTTPS (set true to enable self-signed cert — required for GPS on mobile)
FLASK_HTTPS=false

# AgriBot (Groq API key — get one free at https://console.groq.com)
GROQ_API_KEY=your-groq-api-key-here
```

### 3. Launch
**On Windows — just double-click:**
```
start.bat
```
This will automatically:
- Create a virtual environment (`.venv`) if it doesn't exist
- Activate it
- Install all dependencies from `requirements.txt`
- Call **`python run.py`** to start the server

> `start.bat` is just a setup wrapper — it always ends by running `run.py`.
> You can also run `python run.py` directly once the venv is already set up.

**Or manually (first time or any time):**
```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
python run.py
```

### 4. Open in browser
```
http://localhost:5000
```

> The database (`agrisearch`) and all required tables are **created automatically** on first run via `db.create_all()`. No migration tools needed.

## 💻 Running on a New / Different Device

Follow these steps on any Windows PC that does not yet have the project set up.

### Step 1 — Install Python
1. Download Python 3.8+ from [python.org/downloads](https://www.python.org/downloads/)
2. Run the installer — **tick "Add Python to PATH"** before clicking Install
3. Verify: open Command Prompt and run:
   ```
   python --version
   ```

### Step 2 — Install MySQL
1. Download **XAMPP** (easiest) from [apachefriends.org](https://www.apachefriends.org/) **or** MySQL Community Server from [dev.mysql.com/downloads](https://dev.mysql.com/downloads/mysql/)
2. Start the **MySQL** service (in XAMPP Control Panel click **Start** next to MySQL)
3. Default credentials: host `localhost`, user `root`, password *(blank)*

### Step 3 — Copy the Project
Copy the entire `agrisearch/` folder to the new device.  
**Do NOT copy** these — they're auto-generated:
```
.venv/          ← will be created by start.bat
__pycache__/    ← auto-generated by Python
*.log           ← log files
```
**Must include:**
```
.env            ← your database & API key settings
requirements.txt
start.bat
run.py
app/
config.py
db_init.py
```

> If sharing via USB/zip, make sure `.env` is included. It may be hidden in Windows Explorer — enable "Show hidden files" in Folder Options.

### Step 4 — Create the `.env` file (if not copied)
In the project root, create a file named exactly `.env` (no extension) with:
```env
FLASK_APP=run.py
FLASK_ENV=development
SECRET_KEY=change-this-in-production

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=agrisearch

API_PORT=5000
FLASK_HTTPS=false
GROQ_API_KEY=your-groq-api-key-here
```

### Step 5 — Run the System
Double-click **`start.bat`**.

It will:
1. Create the Python virtual environment
2. Install all packages from `requirements.txt`
3. Auto-create the `agrisearch` database in MySQL
4. Auto-create all tables
5. Start the server at `http://localhost:5000`

## 🌐 Accessing from Another Device on the Same Network

To let phones, tablets, or other computers on the same Wi-Fi access the system:

### On the host PC (where the server runs):
1. Find the host's local IP address:
   ```
   ipconfig
   ```
   Look for **IPv4 Address** under your active network adapter, e.g. `192.168.1.5`

2. The server already listens on `0.0.0.0` (all interfaces) by default

3. Allow port 5000 through Windows Firewall:
   - Open **Windows Defender Firewall** → Advanced Settings
   - New Inbound Rule → Port → TCP → 5000 → Allow

### On the other device (phone/tablet/PC):
Open a browser and go to:
```
http://192.168.1.5:5000
```
*(replace with the actual IP of the host PC)*

> **For GPS features on mobile:** GPS requires HTTPS. Set `FLASK_HTTPS=true` in `.env`, restart, then open `https://192.168.1.5:5000`. Accept the browser's self-signed certificate warning.

## 🔒 Security

- Passwords hashed with **Werkzeug** (PBKDF2-SHA256)
- Session-based auth with configurable timeout
- Password reset tokens expire after 24 hours (one-time use)
- HTTPS support via `ssl_context='adhoc'` (set `FLASK_HTTPS=true` in `.env`)
- All API endpoints return JSON 401 if unauthenticated

## 🤖 AgriBot Setup

1. Get a free API key at [console.groq.com](https://console.groq.com)
2. Add to `.env`:
   ```env
   GROQ_API_KEY=gsk_...
   ```
3. Restart — the chatbot bubble appears automatically for logged-in users

**AgriBot API endpoints:**
```
GET    /chatbot/conversations              List conversations
POST   /chatbot/conversations              Create new conversation
GET    /chatbot/conversations/<id>         Load conversation + messages
POST   /chatbot/conversations/<id>/ask     Send message, get AI reply
DELETE /chatbot/conversations/<id>         Delete conversation
```

## 🐛 Troubleshooting

| Problem | Solution |
|---------|----------|
| `ModuleNotFoundError` | Run `pip install -r requirements.txt` inside the venv |
| Can't connect to MySQL | Ensure MySQL is running; check `DB_HOST`, `DB_USER`, `DB_PASSWORD` in `.env` |
| Port 5000 in use | Change `API_PORT` in `.env` |
| AgriBot not responding | Check `GROQ_API_KEY` is set and valid in `.env` |
| GPS button greyed out | Enable HTTPS: set `FLASK_HTTPS=true` in `.env` |
| Barangay boundaries wrong | Run `python generate_barangays.py` to regenerate |
